General Tips

  • Always utilize URP (Universal Render Pipeline) in every project.
  • For optimal performance, navigate to Edit -> Project Settings -> Time -> Fixed Timestep and set the value to 0.01333.
  • To modify properties for multiple objects simultaneously, select them using Ctrl + Click.
  • For debugging purposes, employ the Debug.Log("Hello World!"); command.
  • Global variables can be modified directly from the Inspector window.
  • If utilizing real values within the Update() function, remember to multiply the force by Time.deltaTime to ensure consistency across different frame rates. Example: 
    rb.AddForce(0, 0, 50 * Time.deltaTime);
  • For physics implementations in scripts, FixedUpdate() is preferable over Update()
  • To enhance control responsiveness, consider increasing the Drag parameter.

Version Control in Unity

  1. Adjust the version control mode to "Visible Meta Files".
  2. Set the asset serialization mode to "Force Text".
  3. Integration with Azure DevOps for version control is highly recommended.

Unity Definitions

Assets:
All files utilized within a project, including GameObjects, Textures, and audio files.
GameObjects:
Interactive entities encountered by players in a game.

Implementing Realistic Physics in Unity

For a realistic physical experience, two essential components are required:

  1. Box Collider: Responsible for collision detection.
  2. Rigid Body: Manages gravitational forces on an object. To add a Rigid Body component to an object, navigate to: Object -> Add Component -> Physics -> RigidBody

To modify a GameObject's components programmatically using C#:

  1. Instantiate an object corresponding to the desired component's class within your script. Example: 
    public Rigidbody rb;
  2. Save the script and proceed to the Unity editor. Drag the relevant component onto the variable you've just declared in the script.
  3. Once linked, modifications can be made within the script. For instance: 
    
void Update() {
    if (example > example2) {
        rb.useGravity = false;
        rb.AddForce(0, 1000, 1000);
    }
}

Utilizing IntelliSense in Unity

IntelliSense in Unity offers a comprehensive list of available commands and functionalities for an object:

  • Type the object name followed by a dot (.) to activate IntelliSense and view available methods and properties.
  • Hover over a particular function to see its detailed documentation, including required arguments and usage.
  • For unfamiliar data types or to gain further insights into a method, consult the official Unity documentation.

Accessing GameObject Information

The Transform class can be used to retrieve location, rotation, and scale details of a GameObject. Example usage:

public Transform varName;

Following this declaration, drag the target GameObject to the variable via the Inspector window.

Script Intercommunication in Unity

To reference one script from another:

  1. Declare an object of the script type you wish to communicate with. Example: 
    public PlayerMovement movement;
  2. In the Inspector window, drag the desired script onto the variable you've defined.
  3. You can now control and manipulate the referenced script as demonstrated: 
    movement.enabled = false;

Handling Time Delays

There are two primary techniques for introducing time delays within Unity:

  • Automatic Method: Implemented using coroutines. Example:

    
StartCoroutine(zombiesSpawn());
IEnumerator zombiesSpawn() {
    yield return new WaitForSeconds(0.5f);
}
  • Manual Method: By leveraging Time.deltaTime to count the elapsed time between frames. It's important to halt the timer once its purpose has been fulfilled to prevent unnecessary computations. Example:

    
bool stopCounting = false;
float timer = 0;

if (!stopCounting) {
    timer += Time.deltaTime;
}

if (timer > 5) {
    // Execute desired actions
    stopCounting = true;
}
