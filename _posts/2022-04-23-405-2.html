Git Notes


Git Basics

The term "origin" refers to a remote repository, while "master" signifies a local branch.


Quickly Uploading a Folder to a Repository

To quickly upload a folder to a repository, follow these steps:

  1. Create the repository on GitHub.
  2. Execute the following commands:

cd <folder_location>
git init
git add .
git commit -m “Descriptive commit message”
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin master


Commit Message Guidelines

When crafting commit messages, ensure you include:

  1. What was changed.
  2. The reason for the change.
  3. Any potential issues to be wary of in the future.


Common Git Commands

  • git init: Initializes a new Git repository in the current directory.
  • git clone [repository URL] [directory to clone to]: Clones a remote repository to the local machine.
  • git add [file/directory name]: Adds files or directories to the staging area.


Working with Large Files (LFS)

If you need to upload a file larger than 100 MB, utilize Git LFS:

  1. Track the large file(s) with the command: git lfs track "file.extension" or git lfs track "*.extension" for multiple files.
  2. A .gitattributes file will be created. Ensure you commit this file: git add .gitattributes
  3. Commit the large file(s) normally: git add file.extension and git commit -m "commit message"
  4. Push the changes to your repository: git push origin main.


Cloning a Specific Commit

If you aim to clone a repository from a particular commit, use the following methods:


git clone –branch commithash --single-branch git://github.com/user/repo.git old_version

Alternatively:


git clone git://github.com/user/repo.git old_version
cd old_version
git checkout commithash


Additional Git Commands

  • git status: Displays the status of the repository, indicating modified files and changes in the staging area.
  • git log -p: Shows the commit history with differences in each commit. The -n parameter limits the number of commits displayed.
  • git diff [file/directory name]: Illustrates the disparities between the working directory and the repository.
  • git branch -a: Lists all branches, including remote branches, with the current branch highlighted.
  • git checkout [branch/commit name]: Switches to a different branch or commit.
  • git merge [branch name]: Merges changes from one branch into another.
  • git pull [remote repository name] [branch name]: Retrieves changes from a remote repository and merges them into the current branch.
  • git push [remote repository name] [branch name]: Sends changes to a remote repository.
  • git fetch [remote repository name] [branch name]: Acquires changes from a remote repository without integrating them.
  • git remote -v: Provides a list of remote repositories and their URLs, with the -v parameter showing detailed information.
  • git config --global user.name "Your Name" and git config --global user.email "email@example.com": Configures global Git settings, including setting user name and email.


Rectifying Mistakes

  • git revert [commit hash]: Reverts a specific commit, creating a new one with reversed changes.
  • git reset --hard [commit hash]: Resets the repository to a specified commit, discarding subsequent changes.
