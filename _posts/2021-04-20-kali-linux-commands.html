Kali Linux Commands Notes


Kali Linux Commands Notes

Common Commands

  • sudo bash

    Grants superuser privileges for all subsequent commands, eliminating the need to prepend each command with sudo.

  • chmod +x <filename.sh>

    Transforms a shell script into an executable file that can be executed in the kernel.

  • ./<filename>

    Executes an executable shell script.

  • pwd

    Displays the current working directory.

  • ls

    Lists the contents of the current directory. Pressing the tab key while typing a directory name provides suggestions.

  • cd <directory>

    Changes the current working directory to the specified directory.

  • cd ..

    Moves up one directory level from the current location.

  • mkdir <directory>

    Creates a new directory.

  • ctrl + R

    Enables search functionality for previously entered commands, similar to the ctrl + f function.

  • man <command>

    Displays the manual page for a specific command, providing usage instructions.

  • shutdown

    Shuts down the system.

  • mv <file name> <directory path>

    Moves files from one directory to another.

  • rm <file>

    Deletes a file.

  • rm -r <directory>

    Deletes a directory and its contents.

  • clear

    Clears the terminal screen of previous commands and output.

  • apt-get update

    Updates the Kali Linux system.

  • apt install <application name>

    Installs a new application.

  • apt --purge remove <program>

    Uninstalls software. The --purge option also removes configuration files.

  • grep <word> <file>

    Searches for a specific word in a given file and displays the lines where the word appears. This is useful for quickly locating information in large text files.

Network Commands

  • ifconfig <interface> down

    Temporarily disables a specified network interface. For example, ifconfig eth0 down.

  • ifconfig <interface> up

    Re-enables a previously disabled network interface. For example, ifconfig eth0 up.

  • ifconfig

    Displays information about network interfaces.

  • ip addr

    Also provides information about network interfaces.

  • iwconfig

    Displays information specifically for wireless networks.

  • ping

    Verifies that a computer can communicate with another over the internet.

  • arp

    Maps a MAC address to an IP address and vice versa.

  • netstat

    Displays statistics about network activities and informs the user about running TCP and UDP ports.

  • route

    Displays the routing table of the network, which is useful for understanding how data packets will be forwarded between different subnets.

Network Attacks

  • airmon-ng check kill

    Terminates processes that interfere with enabling monitor mode on your network adapter. This is often necessary before performing wireless network attacks.

  • airmon-ng start wlan0

    Switches the network adapter from managed mode to monitor mode, which is essential for capturing network packets.

  • airmon-ng stop wlan0mon

    Switches the network adapter from monitor mode back to managed mode. This is useful when you want to revert to normal network operations.

  • airodump-ng wlan0mon

    Displays all networks detected by your adapter, along with details such as connected devices' MAC addresses. This is useful for selecting a target network.

  • airodump-ng -w <fileName> -c <channelNumber> --bssid <BSSID> <interface>

    Monitors a specific network to capture the 4-way handshake between the network and a connecting client. This is often the first step in cracking a Wi-Fi password.

  • aireplay-ng --deauth <times> -a <BSSID | mac of router> -c <MAC of a client> -D wlan0mon

    This command disconnects a client from their network. If you replace <times> with 0, it will continuously keep them disconnected. The -a flag is mandatory, the -c flag is optional (if not specified, it will disconnect all users of the selected network), and -D specifies the network interface you are using.
