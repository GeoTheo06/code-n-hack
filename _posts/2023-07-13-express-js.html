Initializing an Express App

To create an Express application, follow these steps:

  1. Initialize npm by executing the command: npm init -y
  2. Install Express using the command: npm install express
  3. Create a JavaScript file and execute it using: node <jsFile>
  4. (Optional) Install nodemon as a development dependency: npm install -D nodemon
  5. Run the application using: nodemon <jsFile>

For testing requests, consider using Postman.

HTTP Request and Response

REQUEST (req): Represents the HTTP request sent by the client. It contains details about the client's request, such as the HTTP method, requested resource path, query string parameters, and request body.

RESPONSE (res): Represents the HTTP response sent by the server to the client. It contains details about the response, such as the status code, response body, and headers.

Request Message Syntax


1. Request line
2. Header (with host information)
3. Empty line
4. Optional body (additional information if required)

HTTP Methods

  • GET: Request information from the server.
  • POST: Send information to the server.
  • HEAD: Request the header of the response.
  • PUT: Modify data on the server.
  • DELETE: Remove data from the server.
  • PATCH: Apply partial modifications to a resource.
  • TRACE: Provides a debugging mechanism.
  • OPTIONS: Request permitted communication options for a given URL.
  • CONNECT: Start two-way communication between client and server.

Response Message Syntax


1. Status line
2. Header (with server information)
3. Empty line
4. Optional body (additional information if required)

Status Codes

  • 1xx: Informational
  • 2xx: Successful
  • 3xx: Redirection
  • 4xx: Client error
  • 5xx: Server error

Examples of Status Codes

  • 200: OK, SUCCESS
  • 301: MOVED PERMANENTLY
  • 400: BAD REQUEST
  • 404: NOT FOUND
  • 500: INTERNAL SERVER ERROR
  • 502: BAD GATEWAY

Basic Server Syntax


const express = require('express');
const app = express();

// Define route handlers
app.get('/helloworldpage', function (req, res) {
    res.send('<h1>Hello World</h1>');
});

app.get('/anotherpage', function (req, res) {
    res.send('<h1>This is another page.</h1>');
});

// Specify the listening port
const PORT = process.env.PORT || 6969;
app.listen(PORT, () => console.log(`Server started on port: ${PORT}`));

In the example above, the route handler responds to a GET request to the path /helloworldpage. It sends a response to the client with the message "Hello, World!".

Serving HTML Files

Instead of sending raw HTML strings, it's more practical to serve HTML files directly. This can be achieved using the sendFile method.


app.get('/', (req, res) => {
    res.sendFile(path.join(__dirname, 'public', 'index.html'));
});

Serving Static Files

For serving multiple static files, such as stylesheets, scripts, and images, use the express.static middleware.


app.use(express.static(path.join(__dirname, 'public')));

Sending JSON Data

To send JSON data, create JavaScript objects and send them using the json method. Express will handle the serialization.


const members = [
    // ... (members data)
];

app.get('/api/members', (req, res) => {
    res.json(members);
});

Fetching a Single Member

To fetch a specific member based on their ID, use the following approach:


app.get('/api/members/:idNum', (req, res) => {
    // ... (fetching logic)
});

Middleware Functions

Middleware functions are executed before the final request handler. They can perform tasks such as parsing request data, authentication, and modifying the response.


const logger = (req, res, next) => {
    // ... (logging logic)
    next();
};

app.use(logger);
