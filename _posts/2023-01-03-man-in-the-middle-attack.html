When we perfrom a MiTM attack, we usually want to sniff data from a device connected to a network we have access. These data can be passwords, messages and everything else that goes through the internet. The problem is that these data are often encrypted though we'll see what we can do for this later. Another problem which comes before the encryption problem is that we cannot access the data as the device sends them to the router and the router to the internet. To fix this we will have to do an attack called arp poisoning.

The Address Resolution Protocol (ARP) is a networking protocol used to translate Internet Protocol (IP) addresses into physical addresses, such as Ethernet addresses. Every device on a network has a unique physical address (also known as a MAC address) that is used to identify it on the network. When a device wants to send a packet to another device on the same network, it needs to know the MAC address of the destination device. The ARP protocol is used to resolve IP addresses to MAC addresses. When a device wants to send a packet to another device on the same network, it sends an ARP request to the network, asking for the MAC address of the destination device. If the destination device is on the same network, it responds with its MAC address, and the sender can then use this information to send the packet directly to the destination device.

ARP poisoning (also known as ARP spoofing) is a type of attack in which an attacker sends malicious ARP messages to a network in an attempt to associate the attacker's MAC address with the IP address of a legitimate device on the network. This can allow the attacker to intercept and potentially modify traffic intended for the legitimate device.

Here is an example of how ARP poisoning works:

  1. Attacker A wants to intercept traffic intended for Device B.

  2. Attacker A sends an ARP message to the network, associating its own MAC address with the IP address of Device B.

  3. Devices on the network receive the ARP message and update their ARP caches to associate the IP address of Device B with the MAC address of Attacker A.

  4. When Device C wants to send a packet to Device B, it looks in its ARP cache and sees that the MAC address of Attacker A is associated with the IP address of Device B.

  5. Device C sends the packet to Attacker A, thinking that it is sending it to Device B.

  6. Attacker A receives the packet and can read or modify it before forwarding it to Device B.

What we are going to do to perform the MiTM attack is to set router's ip address to our own MAC address and the device's ip address to our own MAC address as well. Then, the router will send the data meant for the device to me and the device is going to set the data meant for the router to me! To keep the connection alive so that no-one suspects a thing, we are going to forward the messages delivered from the router to the device and the messages delivered from the device to the router! With that set up we are already sniffing the data.
