if you want to create a dependency, run "npm install ". Then, the name and version of the dependency will be listed in the package.json file, and the dependency will be downloaded in the node_modules folder so you can use it later.

The huge npm's advantage is that you can make your project very portable with it. For example, if you are in another computer and need your dependencies, you don't have to download them manually or transfer the node_modules folder, you can simply have th package.json and type "npm install" and it will download all the dependencies that are listed in the package.json.

You can download dev dependencies by typing npm install -D

This is the example of a package.json of a project in which i had installed the uuid dependency and the nodemon as a dev dependency:

{
"name": "nodejs-tutorial",
"version": "1.0.0",
"description": "node crash course",
"main": "index.js",
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"author": "GioTeo",
"license": "ISC",
"dependencies": {
"uuid": "^9.0.0"
},
"devDependencies": {
"nodemon": "^2.0.20"
}
}

from these info, npm can install the needed dependencies to run the project if you don't have them installed.
